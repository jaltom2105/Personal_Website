The Process

This website started as a way to practice building a professional portfolio using AI as a coding partner. Instead of writing every line of code myself, I used Google AI Studio to describe the layout, design, and features I wanted, then refined the results through iteration.

Since it started, the project has grown from a simple static site into a more interactive portfolio with a dedicated "lab" for experiments and a sitewide dark mode system.

Recent Additions

  • The Laboratory Concept: Implemented a modular "Experiments" grid that acts as a sandbox for high-interaction prototypes.
  • Interactive Simulation: Developed Sky Hopper, a custom-built arcade experience using Canvas API designed through purely descriptive prompting.
  • Reactive Design Logic: Engineered a site-wide Dark/Light mode system with persistent state, integrated into a unified CSS variable architecture.
  • Human-Centric Navigation: Refined the information architecture, introducing hierarchical dropdowns and optimized user flows (e.g., the Resume and Lab transitions).

Adversarial UI Testing

One of the most unique aspects of this project is the exploration of "Forbidden UI"—the Don't Click Me experiment. This module tests human curiosity through cautionary visual cues, utilizing immediate browser-level termination commands as a high-stakes interaction model.
